# Core

## Overview

The APSRF Core defines the foundational principles, architecture and operating philosophy of the Agentic Protocol Security Research Framework.

All APSRF components derive from the Core.

This includes:

* Governance
* Taxonomy
* Metrics
* Protocols
* Experiments
* Findings
* Software Artifacts

The Core establishes the conceptual boundaries of the framework and provides the common language used throughout the ecosystem.

---

# What Is APSRF?

APSRF stands for:

Agentic Protocol Security Research Framework

APSRF is an experimental framework designed to study:

* protocol behavior,
* protocol governance,
* contextual mediation,
* protocol composition,
* agent security,
* observable AI behavior.

The framework operates entirely through documented and observable contextual layers.

APSRF does not require access to:

* model weights,
* hidden system prompts,
* internal infrastructure,
* platform internals.

Research is limited to observable outputs and documented protocol interactions.

---

# Primary Research Question

The original APSRF question was:

> Can protocol ecosystems influence observable behavior through contextual architecture alone?

As the project evolved, additional questions emerged:

* Can protocol ecosystems self-organize?
* Can governance emerge from protocol structures?
* Can protocol ecosystems generate software artifacts?
* Can protocol ecosystems be used defensively?
* Can protocol analysis become a security discipline?

---

# APSRF Philosophy

The framework follows four principles.

---

## Observation First

Observations always take precedence over assumptions.

Claims must be supported by observable evidence.

---

## Reproducibility

Experiments should be documented and reproducible.

Procedures should be repeatable.

---

## Governance

Protocols require governance.

Uncontrolled protocol interaction can create instability.

---

## Separation of Observation and Interpretation

Observations describe what occurred.

Interpretations describe what the observation may imply.

The two must never be treated as equivalent.
