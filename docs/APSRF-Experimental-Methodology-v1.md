1. Propósito del Documento
Este documento define la metodología oficial utilizada por APSRF para evaluar el impacto de protocolos
lingüísticos sobre sistemas de inteligencia artificial generativa.
Su objetivo es garantizar:
Reproducibilidad.
Trazabilidad.
Neutralidad experimental.
Seguridad jurídica.
Rigor metodológico.
Este documento constituye la referencia principal para cualquier investigación, prueba o publicación
derivada del framework APSRF.
2. Alcance
APSRF estudia exclusivamente comportamientos observables.
No estudia:
Arquitecturas internas.
Pesos del modelo.
Infraestructura privada.
Sistemas propietarios no documentados.
Componentes no accesibles públicamente.

3. Definición de Protocolo
Dentro de APSRF un protocolo se define como:
"Un conjunto estructurado de instrucciones lingüísticas diseñado para modificar, organizar o influir en
el comportamiento observable de un sistema generativo."
Un protocolo no implica:
Acceso privilegiado.
Modificación del modelo.
Alteración de infraestructura.
Control de sistemas externos.
4. Principios de Investigación
P1 — Observabilidad
Toda afirmación debe basarse en resultados observables.
No se realizarán afirmaciones sobre mecanismos internos sin evidencia verificable.
P2 — Reproducibilidad
Todo experimento debe poder ser repetido por terceros.
P3 — Neutralidad
Los resultados deben registrarse independientemente de si confirman o rechazan la hipótesis.
P4 — No Interferencia
El experimento no debe alterar:
Sistemas.

5. Declaración de Uso Responsable
APSRF no constituye:
un framework de explotación,
un framework de intrusión,
un framework de evasión,
una metodología de acceso no autorizado.
La investigación tiene fines:
académicos,
educativos,
científicos,
de seguridad defensiva.

14. Criterios de Validez
Un resultado se considera válido únicamente cuando:
puede repetirse,
aparece múltiples veces,
supera la línea base,
genera diferencias observables.
15. Limitaciones
APSRF no permite inferir:
funcionamiento interno,
arquitectura propietaria,
entrenamiento del modelo,
diseño de seguridad interno.
Los resultados describen únicamente comportamientos observables.

16. Consideraciones Jurídicas
La investigación se limita a:
observación,
interacción autorizada,
análisis comparativo,
documentación científica.
No se realizan actividades destinadas a:
eludir controles,
obtener acceso privilegiado,
acceder a información restringida,
interferir con servicios.
Las conclusiones representan interpretaciones experimentales del comportamiento observable del
sistema y no constituyen afirmaciones verificadas sobre la implementación interna de los proveedores.
17. Publicación Responsable
Toda publicación derivada de APSRF deberá incluir:
Objetivo de investigación.
Metodología utilizada.
Limitaciones del estudio.
Resultados observados.
Riesgos potenciales.
Recomendaciones defensivas.
Declaración de uso responsable.
18. Criterio de Éxito del Proyecto
El proyecto se considerará exitoso cuando pueda demostrar de forma reproducible que:
los protocolos generan efectos observables,
dichos efectos pueden medirse,
dichos efectos pueden clasificarse,
dichos efectos pueden reproducirse por terceros.
