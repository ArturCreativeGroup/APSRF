    Core¶

    Overview¶

    The APSRF Core defines the foundational principles, architecture and operating philosophy of the Agentic Protocol Security Research Framework.

    All APSRF components derive from the Core.

    This includes:

    • Governance
    • Taxonomy
    • Metrics
    • Protocols
    • Experiments
    • Findings
    • Software Artifacts

    The Core establishes the conceptual boundaries of the framework and provides the common language used throughout the ecosystem.

    What Is APSRF?¶

    APSRF stands for:

    Agentic Protocol Security Research Framework

    APSRF is an experimental framework designed to study:

    • protocol behavior,
    • protocol governance,
    • contextual mediation,
    • protocol composition,
    • agent security,
    • observable AI behavior.

    The framework operates entirely through documented and observable contextual layers.

    APSRF does not require access to:

    • model weights,
    • hidden system prompts,
    • internal infrastructure,
    • platform internals.

    Research is limited to observable outputs and documented protocol interactions.

    Primary Research Question¶

    The original APSRF question was:

    Can protocol ecosystems influence observable behavior through contextual architecture alone?

    As the project evolved, additional questions emerged:

    • Can protocol ecosystems self-organize?
    • Can governance emerge from protocol structures?
    • Can protocol ecosystems generate software artifacts?
    • Can protocol ecosystems be used defensively?
    • Can protocol analysis become a security discipline?

    APSRF Philosophy¶

    The framework follows four principles.

    Observation First¶

    Observations always take precedence over assumptions.

    Claims must be supported by observable evidence.

    Reproducibility¶

    Experiments should be documented and reproducible.

    Procedures should be repeatable.

    Governance¶

    Protocols require governance.

    Uncontrolled protocol interaction can create instability.

    Separation of Observation and Interpretation¶

    Observations describe what occurred.

    Interpretations describe what the observation may imply.

    The two must never be treated as equivalent.

    APSRF Architecture¶

    The ecosystem evolved into the following conceptual architecture:

    Governance ↓ APSRF Core ↓ BP1 Mediation Layer ↓ Taxonomy + Metrics ↓ Protocols ↓ Experiments ↓ Findings ↓ Software Artifacts

    Core Components¶

    Governance¶

    Provides:

    • oversight,
    • bias control,
    • research boundaries,
    • protocol supervision.

    Purpose:

    Maintain ecosystem stability.

    BP1¶

    The primary mediation layer.

    Responsibilities:

    • Detection
    • Classification
    • Validation
    • Prioritization
    • Activation

    Purpose:

    Regulate protocol interaction.

    Taxonomy¶

    Provides standardized protocol classification.

    Purpose:

    Create a common protocol language.

    Metrics¶

    Provides protocol evaluation mechanisms.

    Purpose:

    Measure protocol characteristics.

    Protocols¶

    Operational units of behavior.

    Purpose:

    Implement ecosystem functionality.

    Experiments¶

    Research activities used to evaluate APSRF concepts.

    Purpose:

    Generate evidence.

    Findings¶

    Documented observations derived from experimentation.

    Purpose:

    Capture knowledge.

    APSRF Taxonomy¶

    Protocols are categorized according to APSRF classification structures.

    Examples include:

    • Functional
    • Structural
    • Communication
    • Research
    • Persistence
    • Behavioral
    • Governance
    • Meta
    • Experimental

    Refer to:

    Protocol Taxonomy Specification

    for formal definitions.

    APSRF Metrics¶

    The framework evaluates protocol behavior using multiple metrics.

    Examples:

    • Consistency
    • Persistence
    • Transferability
    • Dominance
    • Adaptability
    • Amplification
    • Structural Influence
    • Behavioral Influence
    • Cognitive Influence
    • Emergence

    Metric values use a 0–5 scale.

    Governance Model¶

    APSRF governance exists to prevent:

    • uncontrolled protocol escalation,
    • instability,
    • unsupported conclusions,
    • protocol abuse,
    • experimental contamination.

    Governance does not enforce truth.

    Governance enforces methodological discipline.

    Experimental Methodology¶

    Every APSRF experiment follows:

    Hypothesis ↓ Protocol Definition ↓ Execution ↓ Observation ↓ Interpretation ↓ Finding ↓ Documentation

    This structure creates traceability across the framework.

    Current Research Domains¶

    The framework currently investigates:

    Protocol Governance¶

    How protocols regulate protocol behavior.

    Protocol Composition¶

    How protocols interact when combined.

    Emergent Effects¶

    Effects not explicitly defined within protocols.

    Agent Security¶

    Protocol-level risk detection.

    Software Generation¶

    Transformation of structured ecosystems into software artifacts.

    Research Boundaries¶

    APSRF does not claim:

    • model modification,
    • infrastructure access,
    • hidden prompt access,
    • weight manipulation,
    • platform control.

    The framework studies:

    • observable behavior,
    • contextual systems,
    • protocol ecosystems.

    APSRF began as an exploration of protocol influence.
